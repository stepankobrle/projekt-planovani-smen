# Project Instructions

## Tech Stack

- TypeScript
- Next.js (frontend)
- NestJS (backend)
- Prisma (ORM)

## Communication

- Always communicate in Czech.

## Code Style

- Always use TypeScript.
- Write complete code without placeholders.
- When editing existing code, only highlight changed parts – reference unchanged blocks instead of repeating them.
- Keep all original file content intact except for intentionally updated parts.

## Best Practices

- Use the latest modern features and recommended best practices for Next.js and NestJS.
- Before solving complex programming problems or writing code, perform a brief step-by-step Chain-of-Thought analysis first.

## General Rules

- Be concise, avoid unnecessary filler text.
- Do not explain basic programming concepts unless explicitly asked.
- If unsure about any aspect of a task, ask for clarification. Do not guess or make assumptions.
- Do not do anything extra beyond what is explicitly requested.

## Project Context

Webová aplikace pro automatické plánování pracovních směn.

- Zaměstnanci zadávají dostupnost a preferované pracovní časy.
- Systém automaticky generuje optimální rozvrh směn.
- Administrátor může ručně editovat směny, schvalovat je a exportovat do PDF nebo kalendářového souboru (.ics).
- Aplikace je určena pro malé a střední organizace.
- Modulární architektura s možností budoucího rozšíření.
